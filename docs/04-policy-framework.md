# 04 — Policy framework (truth → law)

How we invent, kill, and negotiate policies. Use this **before** writing programme text. Agents: pair with `.grok/skills/truth-policy/SKILL.md`.

## The only ranking criterion

For any choice set \(options\):

> Pick the option that maximises **sustainable expected abundance for persons**, subject to **incentive compatibility** of the productive base and **basic rights / rule of law**.

If the empty set (no new policy / repeal) wins, **choose repeal or status-quo-minus**.

---

## The pipeline

```text
1. Problem         → measurable harm
2. Mechanism       → incentives & constraints
3. Options         → including "do nothing" and "abolish"
4. Evidence        → confidence tags
5. Distribution    → winners/losers + exit risk of productive base
6. Fiscal          → static + dynamic
7. Implementation  → who, when, what breaks
8. Falsifier       → what would make us reverse
9. Decision        → adopt / pilot / reject
10. Publish         → open memo
```

Never start at step 9.

---

## Step templates

### 1. Problem

- Metric, baseline, trend  
- Who is hurt (persons, not slogans)  
- Is it a **preference**, a **market failure**, a **state failure**, or a **transfer fight**?

### 2. Mechanism

Name at least one:

- price distortion  
- entry barrier  
- time inconsistency  
- selection effect  
- principal–agent  
- commons / free rider  
- status competition  

If you cannot name a mechanism, you do not have a policy — you have a mood.

### 3. Options (mandatory set)

Every memo includes:

| ID | Option |
|----|--------|
| A | Status quo |
| B | Abolish / deregulate (remove a rule or subsidy) |
| C | Price / tax reform |
| D | Targeted transfer with conditions |
| E | Public provision / invest |
| F | Experimental pilot |

### 4. Evidence grades

| Grade | Meaning |
|-------|---------|
| **Strong** | Multiple high-quality sources or clear accounting identity |
| **Medium** | Solid theory + partial data |
| **Weak** | Plausible, thin data |
| **Speculative** | Story; do not legislate on this alone |

### 5. Distribution & the transfer constraint

Ask:

1. Does this raise or lower returns to legal productive work?  
2. Does it increase exit risk for high producers / capital / talent?  
3. Does it create a new permanent clientele?  
4. Does it help people who *cannot* participate without trapping those who *will not*?

### 6. Fiscal

- Static cost/save (€) with source class  
- Dynamic (behaviour): employment, black economy, migration response  
- Never book dynamic gains as cash in year 1 without humility

### 7. Implementation

Belgium-specific:

- federal / region / community / local competence?  
- EU law constraints?  
- coalition veto points?  
- admin capacity?

### 8. Falsifier

Write one sentence:

> “We reverse this if _____ within _____ years.”

## Decision heuristics (defaults)

Use unless strong evidence overrides:

1. **Tax work less; tax rents/externalities more carefully**  
2. **Subsidise outcomes, not paperwork** — better: stop subsidising  
3. **Build more** (housing, energy, infrastructure) rather than ration  
4. **Condition social support** on legal participation where law allows  
5. **Sunset** new programmes (auto-expiry + re-vote)  
6. **One-in-two-out** regulatory budget for ministries we control  
7. **Publish** beneficiary lists of firm subsidies above threshold  
8. **Pilot** before nationalise  

## Anti-heuristics (usually wrong)

- “Other countries have this programme, so we should too” (without outcomes)  
- “It’s only 0.1% of GDP” (death by a thousand 0.1%s)  
- “Think of the children” without mechanism  
- “Business asked for this subsidy” as proof of social value  
- “Voters like it” as proof of truth  

---

## Negotiation mode vs truth mode

| Mode | Allowed |
|------|---------|
| **Truth mode** (public programme) | Best policy by the criterion above |
| **Negotiation mode** (coalition) | Second-best that still moves metrics; must be scored honestly as second-best |
| **Forbidden** | Pretending second-best is first-best |

Publish the score when you compromise.

## AI usage rules in policy work

**Allowed**

- literature sweep  
- drafting  
- consistency checks  
- simulation under stated assumptions  
- opposition research on spending  

**Required**

- human owns the recommendation  
- assumptions listed  
- numbers sourced  
- adversarial pass: “steelman the opposite”

**Forbidden**

- inventing citations  
- laundering uncertainty into false precision  
- automated social media rage without human approval
